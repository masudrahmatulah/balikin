## 1. Fitur Khusus (Unique Value Proposition)
Agar user mau pindah dari gratis ke stiker berbayar, Anda harus memberikan fitur digital yang tidak ada di versi gratis:

* **Multi-Item Management:** 1 lembar stiker biasanya isi 6-10 QR. User bisa menamai tiap stiker berbeda (Helm, Laptop, Sepeda) dalam satu dashboard.
* **WhatsApp Scan Alert:** Notifikasi instan ke WA pemilik begitu stiker di-scan (Versi free hanya via email/dashboard).
* **Verified Badge (Silver):** Menampilkan centang perak di profil publik untuk meningkatkan kepercayaan penemu.
* **30-Day Scan History:** Melihat riwayat scan (kapan dan di mana) selama 30 hari terakhir.
* **Durability Guarantee:** Penjelasan teknis bahwa stiker ini *waterproof* dan *anti-UV* (tidak pudar kena matahari).

## 2. Alur User & Alur Aplikasi (Sticker Edition)

### **A. Alur Pembelian (Order Flow)**
1.  **Landing Page:** User memilih "Sticker Pack (Isi 6)".
2.  **Checkout:** User isi alamat kirim & bayar via **QRIS** (Midtrans/Xendit).
3.  **Admin Action:** Anda mencetak stiker dengan 6 QR unik yang sudah di-generate sistem namun statusnya masih `unclaimed`.
4.  **Delivery:** Barang dikirim ke user.

### **B. Alur Aktivasi (Activation Flow)**
1.  **Penerimaan:** User menerima stiker, menempelkan satu di Helm.
2.  **Scan Pertama:** User scan stiker di helm tersebut.
3.  **Sistem Cek:** Aplikasi mendeteksi tag `unclaimed` -> Minta User Login.
4.  **Binding:** Setelah login, user menamai tag tersebut: "Helm KYT Merah".
5.  **Status Active:** Sekarang tag tersebut resmi milik user dan berstatus **Normal**.

### Penjelasan Detail Gambar & Panduan untuk Vendor:

Gambar di atas adalah panduan visual untuk vendor cetak Anda. Berikut adalah detail teknis yang perlu Anda sampaikan kepada mereka:

#### 1. Spesifikasi Dokumen (Layout):
* **Ukuran Lembar:** A3 (297mm x 420mm).
* **Mode Warna:** Wajib **CMYK** (agar warna biru Balikin tidak berubah saat dicetak).
* **Resolusi:** Minimal **300 DPI** (agar QR code tajam dan mudah di-scan).

#### 2. Spesifikasi Stiker Individual (Dalam Gambar):
* **Bentuk:** Lingkaran (Bulat).
* **Diameter Potong:** **40mm (4 cm)**. Ini ukuran ideal untuk helm, laptop, atau koper.
* **Jarak Antar Stiker (Bleed & Spacing):**
    * Berikan **Bleed 2mm** (area warna yang lebih luas dari garis potong) agar jika potongan mesin sedikit meleset, tidak ada garis putih di pinggir stiker.
    * Jarak antar garis potong minimal **3mm** agar mesin *cutting plotter* bisa bekerja dengan rapi.

#### 3. Isi Konten Stiker (Wajib Sesuai Gambar):
Setiap stiker dalam lembar A3 tersebut harus berisi 3 elemen utama:
1.  **Logo Balikin:** Di bagian atas.
2.  **Unique QR Code:** Di bagian tengah. **PENTING:** Vendor harus bisa mencetak **QR Code yang berbeda-beda** di setiap stiker dalam satu lembar tersebut (Sistem Variable Data Printing).
3.  **Teks CTA (Copywriting):** Di bagian bawah dengan tulisan yang jelas: **"SCAN SAYA JIKA MENEMUKAN BARANG INI"**.

#### 4. Finishing & Material:
Sampaikan ke vendor spesifikasi ini:
* **Bahan:** Vinyl Putih (Waterproof/Anti Air).
* **Finishing:** Laminasi **Doff (Matte)** atau **Glossy**. Laminasi wajib ada agar QR Code tidak pudar terkena matahari atau goresan di helm.
* **Metode Potong:** **Kiss-Cut (Setengah Putus)**. Stiker dipotong bentuk bulat, tapi lembar A3-nya tetap utuh. User tinggal mengelupas stikernya saja.

#### 5. Registration Marks (Tanda Potong):
Perhatikan tanda siku di keempat sudut gambar mockup. Ini adalah **Registration Marks** (Crop Marks) yang **WAJIB ADA**. Mesin potong vendor (seperti Jinka atau Cameo) membutuhkan tanda ini untuk membaca posisi stiker secara presisi. Tanyakan kepada vendor template *Registration Marks* yang mereka gunakan.

1.  **Generate QR Bulk:** Anda perlu membuat fungsi di backend Next.js untuk men-generate 48 slug unik dan mengubahnya menjadi 48 file gambar QR Code (.png/.svg) sekaligus.
2.  **Layouting:** Gunakan software seperti Adobe Illustrator atau CorelDRAW untuk menata 48 desain stiker tersebut ke dalam template A3 vendor, memastikan setiap stiker memiliki QR Code yang berbeda.
3.  **Kirim ke Vendor:** Kirimkan file PDF/X-1a resolusi tinggi tersebut ke vendor.

Dengan mengikuti panduan di atas, Anda akan mendapatkan produk "Sticker Pack (Isi Banyak)" yang terlihat sangat profesional dan siap dijual di Shopee.

Untuk menghasilkan lembar cetak **Sticker Pack (A3)** dengan puluhan QR Code yang berbeda secara otomatis (bukan *copy-paste* manual satu per satu), Anda membutuhkan kombinasi *stack* untuk **Variable Data Printing (VDP)**.

Berikut adalah *tech stack* yang direkomendasikan untuk proses otomasi desain cetak ini:

### 1. Backend: Data & QR Engine
Ini adalah bagian yang menghasilkan "nyawa" dari setiap stiker (ID unik dan gambar QR-nya).

* **`nanoid` atau `uuid`**: Untuk men-generate *slug* unik yang pendek dan aman untuk URL.
* **`qrcode` (npm package)**: Library paling stabil di Node.js untuk mengubah *slug* menjadi buffer gambar QR Code. Pastikan diatur ke **High Error Correction (Level H)** agar tetap bisa di-scan meski stiker sedikit lecet.
* **`Sharp`**: Ini adalah library wajib. **Sharp** digunakan untuk melakukan *compositing* (menumpuk gambar). Contoh: Menumpuk logo Balikin di atas QR Code, lalu menempelkannya di atas background stiker.

### 2. Layouting: PDF & Vector Generator
Karena vendor cetak membutuhkan file yang tidak pecah (*vector* atau *high-res PDF*), Anda butuh library untuk menyusun grid A3 secara presisi dalam satuan milimeter (mm).

* **`PDFKit`**: Library terbaik untuk membuat PDF dari sisi server. Anda bisa mengatur ukuran kertas ke A3 (297x420mm) dan meletakkan gambar di koordinat X dan Y yang presisi.
* **`Lucide React`**: Jika Anda butuh ikon tambahan di dalam desain stiker tersebut.
* **`Canvas` (node-canvas)**: Jika Anda lebih nyaman menggambar layout menggunakan sintaks HTML5 Canvas sebelum diubah menjadi file gambar atau PDF.

### 3. Admin UI: Dashboard Operasional
Anda butuh satu halaman khusus (Admin Only) di aplikasi Next.js Anda untuk memicu proses ini.

* **`shadcn/ui` (Table & Button)**: Untuk menampilkan daftar pesanan stiker yang masuk.
* **Server Actions (Next.js)**: Untuk menjalankan proses berat (generate 48 QR -> Compose -> Generate PDF) di sisi server agar tidak membebani browser user.

### Contoh Logika Sederhana (Node.js + Sharp)

Jika Anda memilih **Full Auto**, bayangan kodingannya akan seperti ini:

```typescript
// Script singkat bayangan prosesnya
import sharp from 'sharp';
import QRCode from 'qrcode';

async function generateSticker(slug: string) {
  // 1. Generate QR Code
  const qrBuffer = await QRCode.toBuffer(`https://balikin.id/p/${slug}`);

  // 2. Gabungkan dengan Template Background menggunakan Sharp
  const finalSticker = await sharp('template-stiker-kosong.png')
    .composite([
      { input: qrBuffer, top: 100, left: 100 }, // Posisi QR di tengah
      { input: 'logo-balikin.png', top: 20, left: 100 } // Posisi logo di atas
    ])
    .toBuffer();

  return finalSticker;
}
```
